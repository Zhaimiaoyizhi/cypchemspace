cypchemspace 课程项目总需求文档

本文档可直接在浏览器中编辑正文区域，用于锁定课程项目定位、边界和验收口径。

1. 项目定位

将 CYP_comparison 中最适合课程展示的一条主线包装为 Python package：读取小型底物表，计算 RDKit Morgan fingerprint，生成 UMAP/PCA 二维化学空间图，并用 kNN permutation test 检验 mydb 底物相对 P450DB 背景的局部富集。
挑刺：项目不应声称复现完整科研仓库；课程目标是 package、API、CLI、测试、notebook 和可复现 demo。

2. 提交边界

提交 course_project_cypchemspace 独立目录。只附带教学级小样例数据，不提交完整 mydb/P450DB 大表，不依赖在线 PubChem 请求。
挑刺：如果后续想提交全量结果，应只给下载/复现说明或摘要表，避免把大数据和历史输出混进课程包。

3. 核心用户与场景

面向任课教师和助教。评审者应能在本地安装包、运行 CLI、打开 notebook，并理解生物学问题、软件结构、统计方法和局限。

4. 验收标准

满足 pip install -e .、3-5 个可复用 API、一个 CLI、至少 5 个 pytest、demo notebook、README、核心图表、environment.yml/requirements.txt、课程报告和 slides outline。
挑刺：UMAP 图不能单独支撑结论；报告中必须同步解释 kNN permutation statistic。

5. 不做事项

不包装 LLM rescue v2 为主功能；不重跑所有 fingerprint sensitivity；不把 logP 或红框蛋白判断写成主流程硬依赖；不承诺膜定位证明。
